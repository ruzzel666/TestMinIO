# Интеграция ASP.NET Core с S3-совместимым хранилищем MinIO

В данном проекте реализовано хранение и обработка файлов с использованием self-hosted S3-совместимого объектного хранилища **MinIO** и **ASP.NET Core Web API**. 

Работа демонстрирует архитектурный подход по отделению хранения тяжелых бинарных данных от прикладной логики приложения. Это позволяет сделать основные сервисы полностью stateless, что критически важно для горизонтального масштабирования и отказоустойчивости.

## 🛠 Стек технологий
* **Backend:** ASP.NET Core Web API, C#
* **S3 Client:** Minio SDK (v6.0.0+)
* **Инфраструктура:** Docker, Docker Compose
* **Документация API:** Swagger / OpenAPI

## 🏗 Архитектура решения
Взаимодействие компонентов выстроено по следующей схеме:
`HTTP Client (Swagger) → ASP.NET Core API → MinIO (S3 API) → Bucket / Objects`

Логика работы с S3 API инкапсулирована в выделенном слое `FileStorageService`, что обеспечивает чистоту контроллеров и удобство поддержки.

## 🚀 Реализованный функционал
В рамках проекта контроллер `FilesController` предоставляет следующие endpoint'ы:
1. **`POST /api/files/upload`** — Загрузка файла (multipart/form-data) в бакет `files`. Если бакет не существует, он создается автоматически.
2. **`GET /api/files/download/{name}`** — Скачивание файла по его имени (через поток, без загрузки всего файла в оперативную память).
3. **`GET /api/files/list`** — Получение списка всех загруженных объектов в бакете (имя, размер, дата изменения) с использованием асинхронных итераторов (`IAsyncEnumerable`).
4. **`GET /api/files/presigned-url/{name}`** — Генерация временной безопасной ссылки (Presigned URL) на скачивание файла, позволяющей клиенту забрать файл напрямую из MinIO, минуя backend приложения.

## ⚙️ Запуск проекта

### 1. Развертывание инфраструктуры (MinIO)
Хранилище MinIO запускается в Docker-контейнере. Для персистентного хранения данных (чтобы файлы не пропадали после перезапуска) настроен volume.
Выполните команду в корне проекта, где лежит `docker-compose.yml`:
`docker-compose up -d`

*После запуска MinIO Console будет доступна по адресу `http://localhost:9001` (Учетные данные: `minio_access_key` / `minio_secret_key`).*

### 2. Запуск Web API
Запустите ASP.NET Core приложение через вашу IDE или командой:
`dotnet run`

*Swagger UI будет доступен по адресу `http://localhost:<port>/swagger`.*
